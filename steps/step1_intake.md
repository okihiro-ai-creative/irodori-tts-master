📖 必読: `resources/CHARACTER_PRESET_TEMPLATE.md`（後工程へ渡す固定項目を確認するため）

## この工程でやること

ユーザー入力から、用途、台本、キャラクター固定方式、参照音声の有無、出力先、実行するかコマンド生成のみかを確定し、`outputs/intake.md` に保存する。

### 確認する項目

| 項目 | 必須 |
|---|---|
| 台本または台本ファイル | ✅ |
| 用途 | ✅ |
| 希望する声質・キャラクター方向 | ✅ |
| 参照音声の有無 | ✅ |
| 実行環境 | ✅ |
| 出力先 | ✅ |
| 安全確認 | ✅ |

### ファイル操作の可否（原則8）

| 操作 | 可否 |
|---|---|
| `outputs/intake.md` を作成 | ✅ |
| 既存の台本ファイルを上書き | ✗ 禁止（理由: 正本を壊すため） |
| 参照音声を移動・削除 | ✗ 禁止（理由: 元素材の所在と権利確認を失うため） |

### 正例・反例（原則3）

✅ 正例:
```markdown
用途: YouTube解説のナレーション
固定方式: no_ref（caption + seed固定）
希望声質: 落ち着いた若い女性、近い距離感、やわらかい
出力: コマンド生成のみ
安全確認: 実在人物の模倣なし、参照音声なし
```

✗ 反例:
```markdown
用途: なんとなく
声: 有名声優の〇〇っぽく
固定方式: seedだけで固定
```

### この工程固有の禁止事項（原則2）

- ✗ 参照音声の同意確認なしに `ref_wav` を選ぶことを禁止する。なぜなら無断クローンのリスクがあるからだ。
- ✗ 台本が未確定のままcaptionやコマンドを作ることを禁止する。なぜなら読み・尺・演技設計が変わるからだ。
