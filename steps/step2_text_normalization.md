## この工程でやること

Irodori-TTS向けに日本語本文を整形し、`outputs/normalized_text.md` に保存する。難読漢字、固有名詞、英数字、記号、長すぎる文を音声向けに調整する。

### 整形ルール

- **固有名詞・作品固有用語はひらがなに変換する**（誤読すると視聴者に気づかれるため）
- **難読漢字はひらがな・カタカナに直す**
- **誰でも読める一般漢字語は漢字のまま残す**（全文ひらがな化はイントネーションを壊すため禁止）
- 英数字は読み上げたい形に寄せる
- 1文が長い場合は自然な句点で分ける
- 絵文字制御を入れる場合は、狙いをコメントで記録する
- 意味の要約・言い換えはユーザー指定がある場合のみ行う

### ファイル操作の可否（原則8）

| 操作 | 可否 |
|---|---|
| `outputs/normalized_text.md` を作成 | ✅ |
| `outputs/intake.md` を上書き | ✗ 禁止（理由: 前工程の確定値を壊すため） |
| 元台本を上書き | ✗ 禁止（理由: 正本と派生物を分離するため） |

### 正例・反例（原則3）

✅ 正例:
```markdown
本文:
きょうは、Irodori-TTSでキャラクター音声を固定する方法を解説します。

読み注意:
- Irodori-TTS: いろどり ティーティーエス
- VoiceDesign: ボイスデザイン
```

✗ 反例:
```markdown
本文:
今日はIrodori-TTS及びVoiceDesignにおけるseed値の再現性について包括的に...
```

長すぎる文と英字読みが未処理。

### この工程固有の禁止事項（原則2）

- ✗ PowerShellの文字化け出力を根拠に日本語本文を修正することを禁止する。なぜなら誤った文字列で上書きする危険があるからだ。
- ✗ 台本を勝手に短縮することを禁止する。なぜなら音声尺と内容が変わるからだ。
